Vou criar uma estrutura de pastas robusta para o seu projeto CLI `poc-coder-cli`. Aqui está a estrutura que vou implementar:

```
poc-coder-cli/
├── bin/                     # Contém scripts executáveis
├── src/                     # Código fonte principal
│   ├── commands/            # Comandos CLI individuais
│   ├── utils/               # Utilitários e funções auxiliares
│   ├── services/            # Serviços de negócios
│   ├── config/              # Configurações
│   ├── models/              # Modelos de dados
│   └── index.js             # Ponto de entrada principal
├── tests/                   # Testes
│   ├── unit/                # Testes unitários
│   ├── integration/         # Testes de integração
│   └── fixtures/            # Dados para testes
├── docs/                    # Documentação
├── scripts/                 # Scripts de automação/build
├── .github/                 # Configurações de CI/CD para GitHub
│   └── workflows/           # Fluxos de trabalho GitHub Actions
├── .gitignore               # Arquivos a serem ignorados pelo git
├── .eslintrc.js             # Configuração ESLint
├── .prettierrc              # Configuração Prettier
├── jest.config.js           # Configuração Jest para testes
├── package.json             # Dependências e scripts
├── README.md                # Documentação principal
└── LICENSE                  # Licença do projeto
```
